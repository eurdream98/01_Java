스레드

프로세스 : 컴퓨터에서 실행 중인 프로그램 운영체제(OS)에서 제공하는 자원을 할당받으며 프로세스는 독립적이다.

스레드는 프로세스내에서 실행되는 실행 흐름의 단위이다. 

하나의 프로세스에는 여러 개의 스레드가 있을 수 있다. 

스레드를 이용하여 다중 작업을 수행하는 것이 일반적이다.

멀티 프로세스 : 운영체제에서 여러개의 프로세스를 동시에 실행하는 것

멀티 스레드 : 한 개의 프로세스 내부에서 여러개의 스레드가 있는 것

멀티 스레드 : 스레드2개가 병렬적으로 운영됨
싱글 스레드 : 메인 스레드 하나만 가지고 작업을 처리

멀티 스레드 장점
사용자의 입력을 기다리는 구간 내에서 작업을 실행시킬 수 있다. -> 애플리케이션의 응답성 향상

단점
만약 a와 b가 동시에 멀티스레드에서 일어날 때 같은 자원을 써야할 일이 생긴다면 코드에 혼란이 생긴다.(동기화 발생) 자원에 하나의 스레드만 접근하도록 기다리게하는 것을 락을 걸어놨다혹은 동기화처리라고 한다. 
교착상태
각각의 연결된 자원에서 하나씩 자원을 갖고 있고 다음 연결된 자원으로 이동해야하는데 서로 하나씩 갖고 있기 때문에 대기상태에 빠질 수 있다.

모든 자바 프로그램은 메인 스레드가 메인메소드를 실행하며 시작한다. 

메인스레드가 끝나면 가비지컬렉션이 작동할 필요가 없다. 

가비지 컬렉션처럼 보조적인 친구들은 데몬스레드로 정해놓으면 프로그램이 종료되면 같이 강제종료된다

데이터베이스 개요

데이터베이스란 한 조직에 필요한 정보를 여러 응용 시스템에서 공용할 수 있도록 논리적으로 연관된 데이터를 모으고 중복되는 데이터를 최소화하여 구조적으로 통합/저장해놓은 것이다. 구조화된 데이터의 집합이므로 컴퓨터에서 쉽게 접근,관리,업데이트할 수 있다.

특징
1. 실시간 접근성
2. 계속적인 변화
3. 동시 공유
4. 내용에 따른 참조 -> 게시글을 검색할 때 어떤 키워드가 들어가는 데이터를 검색하고 싶다.

DBMS : 데이터를 관리할 수 있는 프로그램,데이터를 추출,조작,정의.제어할 수 있게 해주는 관리 프로그램

Create,Read,Update,Delete ->CRUD

DBMS의 기능
데이터 추출 -> 사용자 조회
데이터 조작 -> 삽입,수정,삭제 작업
데이터 정의 -> 데이터의 구조를 정의하고 데이터의 구조를 삭제,변경 기능
데이터 제어 -> 저장한 데이터를 백업, 오류가 있을 때 회복하던가 동시에 데이터를 다루려고할 때 기능 제공

DBMS 이점
1. 데이터 중복 최소화 -> 데이터와 응용프로그램을 분리시킴으로써 상호 영향정도를 줄일 수 있다. 프로그램이 갑작스럽게 꺼졌을 경우 데이터를 보존할 수 있다.
2. 쿼리언어
3. 데이터의 무결성 -> 무결성이란 결함이 없다는 뜻이다. 데이터에 다양한 제약조건을 걸어 무결성을 유지할 수 있다.
4. 데이터 백업 및 복구 -> 백업,복구 지원 -> 데이터 복원 가능
5. 표준화 -> 어플리케이션 개발 및 유지보수가 유리해짐
